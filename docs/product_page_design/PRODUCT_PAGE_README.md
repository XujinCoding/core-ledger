## 📋 API接口清单

### 分类管理 (9个接口)
- `GET /api/categories/tree` - 获取分类树
- `GET /api/categories` - 获取分类列表
- `GET /api/categories/{id}` - 获取分类详情
- `GET /api/categories/{id}/children` - 获取子分类
- `POST /api/categories` - 创建分类
- `PUT /api/categories/{id}` - 修改分类
- `PUT /api/categories/{id}/status` - 启用/禁用
- `PUT /api/categories/{id}/move` - 移动分类
- `DELETE /api/categories/{id}` - 删除分类

### 商品管理 (6个接口)
- `GET /api/products` - 获取商品列表（支持筛选、分页）
- `GET /api/products/{id}` - 获取商品详情
- `POST /api/products` - 创建商品
- `PUT /api/products/{id}` - 修改商品
- `PUT /api/products/{id}/status` - 启用/禁用
- `DELETE /api/products/{id}` - 删除商品

### 属性管理 (2个接口)
- `GET /api/products/{id}/attrs` - 获取商品属性
- `PUT /api/products/{id}/attrs/batch` - 批量更新属性（新增、修改、删除）

**总计**: 17个接口

**注意**: SKU由后端自动处理，前端无需调用SKU相关接口

## ⚠️ 重要提示

### 业务逻辑
1. **分类限制**: 最大4层，有子分类或商品时不能删除
2. **属性限制**: 最多5个属性，修改后后端自动更新SKU
3. **权限控制**: 仅管理员可访问商品管理功能
4. **商品状态**: 禁用后不可见

### 性能考虑
1. **分类树缓存**: 建议缓存5分钟
2. **图片懒加载**: 列表中使用懒加载
3. **分页加载**: 商品列表分页展示
4. **请求优化**: 防抖、节流

### 用户体验
1. **操作反馈**: 所有操作都要有明确反馈
2. **错误提示**: 友好的错误提示信息
3. **确认对话框**: 危险操作必须二次确认
4. **加载状态**: 显示加载进度和状态
5. **简化流程**: 属性管理集成在商品编辑页，一次保存完成
